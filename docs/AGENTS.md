# Documentation Rules

## Priorities

- `P0`: User-facing correctness and traceable facts.
- `P1`: Stable, present-tense language integrated into the relevant section.
- `P2`: Concise wording and easy navigation.

## Critical Rules

- Document the status quo directly in the relevant section instead of using standalone labels such as `Current status` or `Next step`.
- Keep historical rollout wording only inside explicit history/changelog sections.
- When documenting test architecture, separate facts (`what exists now`) from recommendations (`what to do next`).
- Create dedicated follow-up documents only when real future work remains open.
