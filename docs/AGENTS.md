# Documentation Rules

## Priorities

- `P0`: User-facing correctness and traceable facts.
- `P1`: Stable, status-based language for current behavior and next steps.
- `P2`: Concise wording and easy navigation.

## Critical Rules

- Prefer status-based phrasing (`Current status`, `Next step`) over time-relative rollout wording.
- Keep historical rollout wording only inside explicit history/changelog sections.
- When documenting test architecture, separate facts (`what exists now`) from recommendations (`what to do next`).
- For follow-up work, link to a dedicated follow-up document instead of embedding long future plans inline.
