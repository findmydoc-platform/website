# Assets & media

Stories must be deterministic and work offline. Use local assets and shared fixtures.

## Asset policy

- Do not hotlink images or videos from external URLs.
- Store reusable Storybook media in `src/stories/assets/`.
- Import Storybook media through `src/stories/fixtures/assets.ts` and use semantic fixture exports in stories.
- Placeholders may be committed as PNGs following the `placeholder-<width>-<height>.png` naming pattern.

These rules are enforced by repository-wide Storybook guidance.

## Shared fixtures

Use story fixtures to keep data consistent and reusable:

```ts
export const listingFixture = {
  id: 'listing-01',
  name: 'Atlas Medical Center',
  city: 'Lisbon',
  rating: 4.7,
}
```

## Examples

```tsx
import { storyClinicImages } from '@/stories/fixtures/assets'

const meta = {
  title: 'Domain/Listing/Organisms/Heroes/FeatureHero',
  parameters: { layout: 'fullscreen' },
}

export const Default = {
  args: {
    image: storyClinicImages.landing.hero,
    heading: 'Care abroad, made simple',
  },
}
```

## Tip

If you need a new asset, add it under `src/stories/assets/`, expose it from `src/stories/fixtures/assets.ts`, and import the semantic fixture export in the story.
